## Role Responsibilities per Sprint

### Project Manager / Scrum Master
- Create sprint plan, assign story owners.  
- Run sprint planning, review, retrospective meetings.  
- Update backlog priorities based on stakeholder feedback.

### QA Lead
- Write acceptance tests for each story.  
- Perform regression testing before each release.  
- Document bugs in issue tracker (e.g., GitHub Issues).

### DevOps Lead
- Maintain server (Node.js, Socket.io) on cloud platform.  
- Ensure environment variables (port, API keys) are secure.  
- Automate deployment on push to `main` branch.

### Docs Lead
- Keep `backlog.md`, `sprint-*.md`, `schedule.md` up to date.  
- Document API endpoints (if any) and game rules.  
- Prepare final project report and presentation slides.

### Frontend / UI Lead
- Build and maintain all frontend screens (lobby, waiting, game).  
- Implement drawing canvas, toolbar, chat, and real‑time UI updates.  
- Ensure mobile responsiveness and cross‑browser compatibility.

## Communication
- Daily stand‑up: 15 min via Discord/Slack.  
- Weekly sprint planning: Thursday 3 PM.  
- Tools: GitHub (code + issues), Google Docs (shared notes), Trello (optional task board).
